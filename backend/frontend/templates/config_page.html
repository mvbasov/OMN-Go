Database Backups
When off, the server answers only this device (127.0.0.1 / localhost). When on, it listens on all network interfaces so other devices can connect using the admin or guest password. Takes effect after application restart.
Identifies this device in database backup filenames (e.g. pixel7, desktop). Letters, digits, '_' and '-' only; anything else is replaced. Clearing the field resets it to the OS hostname.
How many backups to keep per database on the Database Backups page; creating a new backup deletes the oldest beyond this count. Default 3.
Limits how large an image or JSON file can be when uploaded from the editor (drag-and-drop) or shared into Quick Notes from Android. Default 3 MB.

Android Integration

Lets a note launch Android "intent:" URIs when tapped in the app - e.g. opening a Settings screen with [Wi-Fi](intent:#Intent;action=android.settings.WIRELESS_SETTINGS;end;). Off by default. Android only; ignored on desktop and in LAN browsers.
Additionally allows notes to run shell commands via Termux (RUN_COMMAND). Requires "Enable intent: links" above to also be on, Termux installed with allow-external-apps=true, and its RUN_COMMAND permission granted; each command asks for confirmation before it runs. Security: notes arriving via git sync or LAN editing could contain commands - leave this off unless you author your own notes.
